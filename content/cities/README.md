# content/cities

## Purpose
Canonical structured packages for city entities (Bangkok and expansion cities).

## Allowed files
- JSON packages: `{city-slug}.json` or `{city-slug}/manifest.json`
- This `README.md`

## Naming convention
- Slug: lowercase kebab-case (`bangkok`, `chiang-mai`, `phuket`)
- Consistent with geography taxonomy and public route segments

## Owner
Content / Data team

## Validation rules
- Must pass DATA_STANDARD / factory schemas
- Required multilingual names and SEO fields when publish-ready
- Sources required for non-obvious factual claims

## Future usage
City landing content source of truth and parent relation for districts/areas.
