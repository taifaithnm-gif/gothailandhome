# MARKETPLACE_FLOW_REVIEW

**Date:** 2026-07-15

## Flows reviewed

| Flow | Route | Outcome |
|------|-------|---------|
| Find My Home | `/en/find-my-home` | Private demand intake; success copy promises non-publish |
| List Your Property | `/en/list-your-property` | Owner submission pending review |
| Developer partnership | `/en/partners/developers` | Application pending review |
| Agency partnership | `/en/partners/agencies` | Application pending review |
| Platform support | `/en/contact` + listing escalation | CS contacts; not listing owners |
| Viewing request | Listing detail | Consent + submit |

## Persona fit

- **Owner / agent / developer:** forms exist; no status portal after submit (email-only expectation — potential dead end).  
- **Buyer/tenant:** Find My Home + viewing request reduce public login need.  
- **Platform help:** Contact page correctly scopes Apple as CS.

## Friction / dead ends

1. Nav puts partnership and list-property beside browse — role confusion.  
2. No knowledge center / FAQ for foreign buyers (articles absent).  
3. Admin exists for staff; not an Alpha public dashboard.  
4. Contact coverage on listings is effectively platform-mediated for all sourced stock.

## Contact-model Alpha promise (recommended messaging)

State explicitly: “We index public listings; we are not the listing agent; we can help contact the source.”
